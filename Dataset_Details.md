# BirdCLEF 2026: 데이터셋 및 증강 기법 상세 문서

이 문서는 BirdCLEF 2026 파이프라인에서 원본 오디오 데이터를 전처리하고 증강하는 전략과, 배치 구성을 위한 데이터 로더 및 교차 검증(K-Fold) 전략을 상술합니다.

## 1. 원본 데이터셋 구조

* **`train_audio/` (Short Clips)**: Xeno-canto 등에서 수집된, 파일 하나당 특정 새의 소리가 주를 이루는 짧은 오디오 클립(.ogg)입니다. 총 234개의 대상 종(Species) 폴더별로 나뉘어 있습니다. 일부 종은 수백 개의 파일이 있지만, 일부 희귀 종은 몇 개의 파일만 존재하는 극심한 데이터 불균형(Imbalance) 상태입니다.
* **`train_soundscapes/` (Long Audio)**: 실제 현장(파나마 습지)에서 방음 기록기로 무인 녹음된 1분 길이의 오디오 파일들입니다. 
* **`train_soundscape_labels.csv`**: `train_soundscapes` 파일들을 5초 단위 구간으로 잘라 각 구간마다 실제로 어떤 새가 울었는지 다중 레이블로 표기해 놓은 정답지 파일입니다. (새소리가 없으면 `nocall`로 기록됨)

## 2. 모델에 입력하기 위한 데이터 전처리 (Modifications)

오디오의 길이가 제각각이므로, 2D CNN에 입력할 수 있도록 크기가 고정된 스펙트로그램 이미지로 변환하는 일련의 과정(`src/utils/audio.py` 및 `src/data/dataset.py`)을 거칩니다.

1. **리샘플링 및 채널 통합**: 모든 오디오를 BirdCLEF 표준 샘플링 레이트인 **32,000Hz**로 로드하고, 다채널 오디오는 **Mono(1채널)**로 평균을 내어 통일합니다.
2. **5초 구간 자르기 (Pad or Trim)**: 
   * 모델의 기본 입력 윈도우 사이즈는 **5.0초 (160,000 샘플)**입니다.
   * `train_soundscapes` 데이터는 1분 오디오를 5초 단위로 잘라서 사용합니다 (끝에서부터 5초 역산 계산 등).
   * `train_audio` 데이터가 5초보다 길면 무작위 부분 또는 앞부분을 자르고(Trim), 5초보다 짧은 파일은 소리를 반복해서 이어 붙여(Wrap padding) 정확히 5초로 맞춥니다.
3. **데시벨 정규화 (Amplitude Normalization)**: 소리의 크기가 천차만별이므로, 파형의 RMS(Root Mean Square) 기준으로 타겟 레벨(`-20.0 dBFS`)이 되도록 볼륨을 평준화합니다.
4. **멜-스펙트로그램(Mel-Spectrogram) 변환**: 
   * 파형 데이터를 `n_fft=1024`, `hop_length=320`, `n_mels=128`, `fmin=50Hz`, `fmax=14,000Hz` 설정의 주파수 스펙트럼으로 변환합니다. (결과 텐서 크기: B x 1 x 128 x 501)
   * 변환된 값은 `[0, 1]` 범위로 Min-Max 정규화하여 이미지처럼 취급합니다.

## 3. 데이터 증강 방식 (Data Augmentation)

새소리의 다양한 백그라운드 노이즈와 변화무쌍한 환경에 모델이 적응할 수 있도록 두 단계에 걸친 강력한 증강 기법(`src/data/augmentation.py`)을 사용합니다.

### 3.1. 파형(Waveform) 단계 증강
1. **가우시안 잡음 추가 (Gaussian Noise)**: `p=0.3`의 확률로 미세한 백색소음 진폭을 섞습니다.
2. **배경음 믹싱 (Background Noise Mix)**: `p=0.3`의 확률로 비, 바람, 곤충 소리 등 임의의 백그라운드 소리를 3~30dB의 SNR(신호 대 잡음비)에 맞춰 원본 파형에 믹싱합니다.
3. **Waveform Mixup**: 
   * 최대 3개의 다른 오디오 샘플을 무작위로 골라와 합치는 기법(`p=0.4`, Dirichlet 분포 `alpha=0.4`)입니다.
   * 한 구간 내에 여러 마리 새소리가 겹치는 현장 상황(Multi-label)을 직접 시뮬레이션하여 일반화 성능을 극대화합니다. 오디오뿐 아니라 정답 라벨(Label)도 가중합을 적용합니다.

### 3.2. 스펙트로그램(Spectrogram) 단계 증강
1. **SpecAugment**:
   * **Time Masking**: 스펙트로그램의 시간 축 특정 구간(최대 20 프레임)을 2군데 무작위로 0(검은색)으로 지웁니다 (`p=0.5`). 찰나의 소리가 누락되어도 맞출 수 있게 합니다.
   * **Frequency Masking**: 스펙트로그램의 주파수 대역 특정 구간(최대 20 멜)을 2군데 무작위로 지웁니다 (`p=0.5`). 특정 음역대가 가려져도 다른 음역 특성으로 새를 판별하도록 강제합니다.
2. **랜덤 타임 롤 (Random Time Roll)**: 스펙트로그램을 시간 축 기준으로 무작위 쉬프트(밀기)합니다 (`p=0.3`). 소리가 정확히 5초의 어느 위치에서 시작하든 무관하게 예측하도록 유도합니다.

## 4. K-Fold 검증 및 데이터 샘플링 방식

대회 리더보드 점수와 100% 비례하는 신뢰도 높은 검증 셋(Validation Set)을 구축하기 위해 다음과 같이 구성합니다.

1. **5-Fold Cross Validation (K-Fold)**:
   * `train_audio` 메타데이터를 기반으로 전체 데이터를 5개의 그룹(Fold 0~4)으로 나눕니다.
   * 4개의 Fold로 학습하고, 남은 1개의 Fold로 검증(Validation AUC 평가)합니다. 학습 시 이 과정을 순차적으로 반복하여 총 5개의 최적 가중치를 만듭니다.
2. **Combined Dataset 결합**: 
   * `train_audio`(단일 클립) 데이터셋과 `train_soundscapes`(현장 녹음) 데이터셋을 PyTorch의 `ConcatDataset`을 통해 하나의 큰 학습 데이터셋으로 합쳐서 학습시킵니다.
3. **역빈도 가중치 샘플링 (WeightedRandomSampler)**:
   * 데이터가 매우 희귀한 조류(예: 샘플 1~2개)와 매우 흔한 조류(수백 개) 간의 불균형을 해결하기 위해 샘플링 확률을 조절합니다.
   * 각 종별 출현 빈도의 역수(`1 / count`)를 가중치로 삼아 데이터 로더가 희귀 종의 데이터를 더 자주 뽑도록 오버샘플링(Oversampling) 합니다.
   * 특히, 타겟 환경과 동일한 도메인인 **`train_soundscapes` 데이터는 평균 가중치의 1.5배 보너스**를 부여하여, 모델이 실전 현장 환경에 더욱 빨리 적응(Domain Adaptation)하도록 설계되었습니다.
